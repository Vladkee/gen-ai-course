# Learn from History Skill

Analyzes past AI session logs to extract lessons learned and materialize them as new or updated agent primitives (skills, instructions).

## When to Use

- **Before starting a new homework** — load lessons from previous homeworks so the same mistakes aren't repeated
- **After finishing a homework** — capture what went wrong while it's fresh
- **When the same problem comes up twice** — a repeated challenge is a signal to encode a fix as a skill

## Procedure

### Step 1 — Collect History

Read the following files (in order, skipping any that don't exist yet):

1. `AI-SUMMARY.md` — cross-homework patterns already identified (read first to avoid re-deriving known issues)
2. `homework-1/AI-CONVERSATION.md` — HW1 session log
3. `homework-2/AI-CONVERSATION.md` through `homework-6/AI-CONVERSATION.md` — per-homework logs

### Step 2 — Extract Patterns

For each file, scan for these signal phrases and build a findings table:

| Signal phrase | What to extract |
|--------------|----------------|
| "Challenges" section | Concrete problem + resolution |
| "failed", "broke", "didn't work" | Failure mode |
| "required several follow-up prompts" | Prompt quality issue |
| "had to manually fix", "missed", "mismatch" | AI error type |
| "Prompts That Worked Well" | Reusable prompt patterns |
| Model change events | Model capability differences |

Output a **Findings Table**:

```markdown
## Findings from History

| Homework | Category | Issue | Resolution Used |
|----------|----------|-------|----------------|
| HW1 | AI error | Field names differed from spec (fromAccountId vs fromAccount) | Cross-check spec JSON vs model after generation |
| HW1 | Process | CLI session lost — no screenshots | Save session immediately after each phase |
| HW1 | Prompt quality | ISO 4217 validation needed multiple follow-up prompts | Provide full requirements in the first prompt |
| ...  | ... | ... | ... |
```

### Step 3 — Classify Each Finding

For each finding, decide which primitive addresses it:

| Finding type | Agent primitive |
|-------------|----------------|
| Recurring process mistake | `.agents/instructions/` — add a rule |
| Reusable prompt pattern | `.agents/prompts/` or skill `assets/` |
| Multi-step procedure | New `.agents/skills/<name>/SKILL.md` |
| One-time correction | Add to existing skill as a "Common Pitfalls" section |

### Step 4 — Propose Changes

Present the user with a list of proposed changes:

```markdown
## Proposed Agent Improvements

### New: `.agents/skills/spec-alignment/SKILL.md`
**Why**: HW1 showed that AI-generated field names drift from spec JSON.
**What**: Skill that cross-checks all model properties against spec sample JSON before finalizing implementation.

### Update: `.agents/instructions/homework-workflow.instructions.md`
**Why**: CLI session loss cost screenshots — Phase 4 verify step needs a screenshot reminder.
**What**: Add "capture screenshots immediately after each demo run" to Phase 4.

### New: `.agents/skills/first-prompt-template/SKILL.md`
**Why**: Partial requirements in the first prompt led to multiple follow-up rounds.
**What**: Skill with a full-requirements prompt template for API scaffolding tasks.
```

Ask: "Should I implement all of these? Or select specific ones?"

### Step 5 — Implement Approved Changes

For each approved change:
1. Create or update the file
2. Note the change in `AI-SUMMARY.md` under "Skills Created" for the relevant homework
3. Add a comment in the created file: `<!-- Generated by learn-from-history skill, [date] -->`

### Step 6 — Update AI-SUMMARY.md

After implementing, append a "Retrospective" entry to `AI-SUMMARY.md` if one doesn't exist for the homework:

```markdown
### Retrospective — [Date]
Skills/instructions created from this analysis:
- [skill/instruction name] — [what it prevents]
```

## Guiding Principles

- **Encode, don't remind** — the goal is a skill that prevents the mistake, not a comment saying "be careful about X"
- **One finding → one primitive** — don't bundle multiple lessons into one file
- **Evidence-based** — only create a new primitive if the same issue appears in ≥1 homework or was explicitly noted as a problem
- **YAGNI** — don't create skills for hypothetical future problems; only address documented ones
