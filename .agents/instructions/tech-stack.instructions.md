# Tech Stack Guidance

## Default Preference

**Backend:** ASP.NET Core 8 (.NET 8)  
**Frontend:** Angular 17+ (standalone components, signals)  
**Test:** xUnit + FluentAssertions + Testcontainers  
**Tooling:** VS Code + .NET CLI + PowerShell scripts

Apply these defaults unless the homework explicitly requires or benefits from a different stack.

## .NET 8 Coding Patterns

### Project Structure
```
src/
  MyProject/
    Controllers/       # Thin controllers, delegate to services
    Services/          # Business logic interfaces + implementations
    Models/            # Domain models (record types preferred for immutable data)
    Validation/        # FluentValidation validators or custom validators
    Program.cs         # Minimal API setup
```

### Model Conventions
```csharp
// Prefer records for DTOs
public record CreateTransactionRequest(
    string FromAccount,
    string ToAccount,
    decimal Amount,
    string Currency,
    TransactionType Type
);

// Use ConcurrentDictionary for in-memory storage
private readonly ConcurrentDictionary<Guid, Transaction> _store = new();
```

### Validation
```csharp
// Data annotations for simple rules
[Required]
[RegularExpression(@"^ACC-[A-Z0-9]{5}$")]
public string AccountId { get; init; }

// Return 400 with structured error
return Results.BadRequest(new {
    error = "Validation failed",
    details = errors.Select(e => new { field = e.Key, message = e.Value })
});
```

### HTTP Status Codes
| Situation | Code |
|-----------|------|
| Created resource | 201 Created |
| Found resource | 200 OK |
| Not found | 404 Not Found |
| Validation error | 400 Bad Request |
| Conflict | 409 Conflict |
| Server error | 500 Internal Server Error |
